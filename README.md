# AmoCRM — Виджет "Недозвон"

## Что делает
- День 3 (рабочих) в этапе "Недозвон" → создаёт задачу ответственному: "Что делаем с лидом?"
- День 5 (рабочих) → автоматически переназначает сделку следующему по очереди из списка
- Список сотрудников управляется через панель настроек прямо в AmoCRM

## Деплой на Render

1. Залей папку `amocrm-widget` на GitHub
2. Создай новый Web Service на render.com
3. Укажи переменные окружения:

```
AMO_DOMAIN=yourcompany.amocrm.ru
AMO_TOKEN=твой_долгосрочный_токен
```

4. После деплоя скопируй URL сервиса (например: https://amocrm-nedozvon.onrender.com)

## Установка виджета в AmoCRM

1. Зайди в AmoCRM → Настройки → Виджеты → Разработчику
2. Загрузи папку `widget` (script.js + manifest.json) как кастомный виджет
3. В настройках виджета укажи URL твоего Render-сервиса
4. В панели виджета добавь нужных сотрудников в очередь и нажми "Сохранить"

## Получить токен AmoCRM

1. AmoCRM → Настройки → Интеграции → Создать интеграцию
2. Получи долгосрочный токен (или используй OAuth)
3. Вставь в переменную AMO_TOKEN на Render

## Проверка работы

- GET https://your-app.onrender.com/ → health check
- GET https://your-app.onrender.com/api/users → текущий список очереди
- POST https://your-app.onrender.com/api/check → ручной запуск проверки

## Расписание проверок
Каждые 2 часа в рабочее время: 9:00, 11:00, 13:00, 15:00, 17:00 (МСК), пн-пт
